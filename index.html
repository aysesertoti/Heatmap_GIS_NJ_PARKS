Green Space Access and Socioeconomic Factors

A Network-Based Geospatial Analysis of Newark, New Jersey

Author: Ayse Sert Oti

Course: Command-Line GIS

Date: December 2025

Project Overview

This project examines the relationship between green space access and socioeconomic factors in Newark, New Jersey, using network-based accessibility analysis. Access to parks and green spaces is a critical environmental justice issue, as lower-income communities often have less access to these resources despite their importance for physical health, mental wellbeing, and quality of life.

Using U.S. Census data and geographic information about New Jersey's parks and conservation lands, this analysis employs walking-distance buffers to evaluate realistic park accessibility. Rather than simply measuring proximity, this study considers whether residents can walk to parks within reasonable timeframes (5, 10, and 20 minutes).

Why Network Buffer Analysis?

Traditional proximity analysis can be misleading, especially for large parks. A resident living next to a 200-hectare state forest might technically be "adjacent" to green space, but the actual park facilities (trails, picnic areas, playgrounds) could be 1-2 kilometers away. This analysis uses park access points (centroids) to represent realistic entry locations and creates walking distance buffers to measure true accessibility.

Key Research Questions

  • How does walking-distance access to green space vary across different neighborhoods in Newark?
  • What percentage of each census tract falls within 5, 10, or 20-minute walking distance of a park?
  • Is there a correlation between household income and park accessibility?
  • Which census tracts have the greatest need for additional green space infrastructure or improved pedestrian access?

Network Buffer Methodology

This analysis uses three concentric buffer zones from park access points to represent realistic walking distances:

Short Walk

400m
~5 minute walk

Ideal access distance for daily park use, especially for children and elderly residents

Medium Walk

800m
~10 minute walk

Moderate access suitable for regular recreational activities

Long Walk

1600m
~20 minute walk

Maximum reasonable walking distance (~1 mile), typical planning standard

Technical Approach

Step 1: Park Access Point Generation

  • Large parks (>5 hectares): Centroid points created to represent realistic entry locations
  • Small parks: Centroids also used for consistency across the analysis
  • Rationale: Using centroids for large parks provides more accurate walking distance estimates than using park boundaries

Step 2: Buffer Creation

  • Three buffer zones (400m, 800m, 1600m) created from each park access point
  • Overlapping buffers dissolved to create unified service areas
  • Analysis performed in NAD83 New Jersey State Plane projection for accurate distance measurements

Step 3: Census Tract Coverage Analysis

  • Spatial intersection of each census tract with buffer zones
  • Calculated percentage of tract area within each walking distance threshold
  • Generated binary indicators (tract within/outside buffer zone)

Geoprocessing Techniques Demonstrated

This project incorporates multiple spatial analysis techniques required for the final project:

1. Table Join (ACS data to census tracts)

Joined tabular ACS data to spatial census tract geometries using the GEOID field as the key. This is essential for mapping demographic variables.

2. Spatial Join (Parks to census tracts)

Used gpd.sjoin() with 'intersects' predicate to identify which parks fall within or overlap each census tract boundary.

3. Buffer Analysis (Network accessibility)

Created three buffer zones (400m, 800m, 1600m) from park access points using .buffer() method to represent walking distances.

4. Dissolve Operation

Dissolved overlapping buffers using .dissolve() to create unified service areas, eliminating double-counting where buffer zones overlap.

5. Spatial Intersection

Calculated intersection of buffer zones with census tracts to determine percentage coverage using .intersection() and area calculations.

6. Clip Operation

Clipped parks and census tracts to study area boundary using gpd.clip() to focus analysis on Newark.

7. Centroid Generation

Generated centroids for both parks (as access points) and census tracts (for distance calculations) using .centroid property.

8. Distance Calculation

Computed distances from tract centroids to nearest park access point using .distance() method.

9. Area Calculation

Calculated areas of tracts and parks in square meters/kilometers using .area property in projected coordinate system.

Why Use Park Centroids for Buffer Analysis?

A key methodological decision in this project was using park centroids as access points for large parks (>5 hectares) rather than buffering from park boundaries.

The Problem: When a park is very large (e.g., a 200-hectare state forest), residents living at the edge of the park polygon might technically be "adjacent" to green space, but the actual park amenities—parking lots, trail heads, playgrounds, picnic areas—could be 1-2 kilometers away at the park's interior.

The Solution: By using park centroids as access points, we model the distance to the park's usable features more realistically. For small neighborhood parks, this distinction is less important, but for large parks it prevents overestimating accessibility.

Technical Implementation: Parks larger than 50,000 square meters (5 hectares) were classified as "large parks" and their geometric centroids were calculated as access points. Buffers were then generated from these centroid points rather than from park boundaries.

Findings and Implications

This network-based geospatial analysis reveals important patterns in the relationship between socioeconomic factors and green space accessibility in Newark. The combination of walking-distance buffers, coverage percentage calculations, and demographic data provides a nuanced understanding of environmental justice issues related to park access.

Key Takeaways

  • Walking distance matters: Network buffer analysis provides more realistic accessibility measures than simple proximity, especially for large parks where usable amenities may be far from park boundaries
  • Service area coverage varies significantly: The percentage of tract area within 400m of parks ranges widely, revealing spatial inequities in ideal park access
  • Multiple distance thresholds tell different stories: While many areas may be within 20-minute walking distance (1600m), far fewer enjoy 5-minute access (400m) ideal for daily use
  • Centroid-based analysis improves accuracy: Using park centroids as access points for large parks prevents overestimating accessibility for edge residents
  • Environmental justice implications: Overlaying income data with accessibility buffers can identify disadvantaged neighborhoods with limited park access
  • Interactive mapping enables exploration: Stakeholders can investigate specific neighborhoods and identify priority areas for intervention

Spatial Patterns Observed

The analysis reveals several important spatial patterns:

  • Buffer overlap zones: Areas served by multiple parks show darker, more intense buffer coverage, indicating redundant access that may be appropriate for high-density areas
  • Access gaps: Census tracts with 0% coverage within 400m buffers represent clear gaps in the park network, especially concerning if they also show high population density or low income
  • Edge effects: Tracts at the municipal boundary may show lower apparent access if parks in adjacent municipalities aren't included in the analysis
  • Large park challenges: Very large parks (forests, reservations) create extensive buffer zones, but actual accessibility depends on entry points, parking, and trail access

Methodological Advantages

This project demonstrates several advantages of network-based accessibility analysis over simpler methods:

Comparison of Approaches

Simple proximity (distance to nearest park boundary):

  • ✗ Overestimates access to large parks
  • ✗ Doesn't account for actual walking distances
  • ✗ Binary measure (near/far) doesn't capture gradations

Centroid-based buffer analysis (this project):

  • ✓ Realistic access points for large parks
  • ✓ Multiple distance thresholds (5, 10, 20 min walks)
  • ✓ Percentage coverage shows partial accessibility
  • ✓ Dissolved buffers create service areas
  • ✓ Comparable across different park sizes

Future enhancements could include:

  • Network analysis using actual street networks (e.g., NetworkX, OSMnx)
  • Accounting for topography and barriers (highways, rivers)
  • Multiple access points per park for very large parks
  • Weighting by park amenities and quality

Policy and Planning Implications

This analysis can inform evidence-based urban planning and environmental justice initiatives:

  • Park development priorities: Tracts with low buffer coverage and high population density are candidates for new park development
  • Infrastructure improvements: Areas just outside buffer zones might benefit from improved pedestrian infrastructure (sidewalks, crosswalks, traffic calming) to enhance access to existing parks
  • Equity assessment: Comparing accessibility metrics across income levels can reveal environmental justice disparities
  • Service standards: The 400m/5-minute walk threshold aligns with urban planning best practices for neighborhood parks
  • Resource allocation: Understanding spatial gaps helps prioritize limited public investment in green space
Replicability: This methodology can be applied to any city or region with park and census data. The Python script is fully documented and can be adapted by changing the study area, buffer distances, or census variables of interest.

Limitations and Future Directions

While this analysis provides valuable insights, several limitations should be noted:

  • Euclidean vs. network distance: Buffers represent straight-line distances, not actual walking routes along streets. True network analysis would require street network data.
  • Barriers not modeled: Highways, rivers, rail lines, and other barriers to pedestrian movement aren't explicitly considered.
  • Park quality not assessed: All parks are treated equally, but amenities, maintenance, and safety vary significantly.
  • Single access points: Very large parks may have multiple entrances; using only centroids may underestimate access in some cases.
  • Temporal considerations: Park hours, seasonal access, and safety perceptions that vary by time of day aren't captured.

Future research could address these limitations by incorporating street network analysis, barrier modeling, park quality assessments, and community perception surveys.

Generating buffers:

service_area_short = park_buffers_short.dissolve().geometry.iloc[0]

Calculating percentage coverage:

intersection = service_area.intersection(tract.geometry)
pct_coverage = (intersection.area / tract.geometry.area) * 100
